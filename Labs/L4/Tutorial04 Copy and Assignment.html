Description:

The purpose of this tutorial is to give you practice writing a proper copy constructor and assignment operator for an entity object class Person and a container class Collection

This tutorial is meant to be started as homework. You will not be able to finish if you only start working on it at your tutorial session.

 

Demo Code

The demo code provides classes Date, EmailAddress, Person, Collection that model person objects who have a name, birthday and an emailAddress. The Collection class provides a container to hold Person objects. The demo code uses the old-style cstrings instead of string objects from the <string> library. The main.cpp file has the test code and currently has the code for the two problems commented out. That is because this code will crash until the required missing parts are added. Study the demo code and run it.

Compile the code with
make clean myApp

and run with
./myApp

Problem 1

Uncomment the Problem 1 portion of the provided main.cpp. Compile and run the code and it will likely crash. The demo version of the Person class relies on the language-provided copy constructor and the language-provided assignment operator. Unfortunately these create memory problems.

For this exercise you need to write your own copy constructor and assignment operator for the Person class. Use those supplied in the EmailAddress class as a starting model and also the examples in the lecture notes on constructors.

Your copy constructor and assignment operator must ensure that no memory leaks or double deletions occur when the test code is run.
