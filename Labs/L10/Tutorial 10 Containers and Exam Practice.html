Description and Demo Code:

The purpose of this tutorial is to have you work with some C++ code that will be the basis of a multipart programming quesiton the final exam. If you do these exercises you will be well prepared for that part of the exam.

Compile and run the demo code provided. All the code is in one file tut10_main.cpp for convenience. Compile it with

g++ -g -Wall tut10_main.cpp

Also review the sections in the course notes on templates and operator overloading to complete the exercises.

 

This tutorial is meant to be started as homework. You will not be able to finish if you only start working on it at your tutorial session.

 

 

Problem 1)

The Bankaccount class currently only has a balance property. Modify the code so that BankAccounts
have a string property owner as well. Modify how BankAccount objects print to, say, Lou: $100 instead of just $100.

 

 

Problem 2)

The Bankaccount class currently implement == and != operators that are based on "identity". That is, the actual addresses of objects are compared. We want to implement these operators as "equality" rather than "identity" operators. Modify the operators so that A == B is true if and only if bankaccounts A and B have equivalent owner strings and have the equal balances. That is, because their owner strings and balances are equal. (Also modify the != operator).

 

 

Problem 3)

Modify the OrderedCollection class so that it behaves like an OrderedSet. The difference is that a set would not be allowed to contain duplicates (based on the == operator of items stored in the set). Here the various add methods need to be changed so that items will not be added if an equivalent item is already in the set. (Note for this exercise you don't actually have to change the name of the class to OrderedSet, though that would be ideal).

 

Problem 4)

Modify the container so that it grows if it gets full. If its internal array gets full a new one should be allocated that is bigger, items copied over, and the old one properly deteted.

 

When you have fixed the three memory leaks show your code to the TA to get credit for the tutorial.

 

Additional Exam Pratice)

Finally the following practice exercise is highly recommended for the final exam:

Take the demo code and remove various method bodies and then re-code them based on their signatures. You should be able to remove the code from any method and re-code it. (On the exam this kind of activity will be done by offering you five alternative implementations of method bodies and asking you to choose the right one.)
